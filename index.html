回溯法

问题

回溯法就是利用递归的思想，用于处理路径之类的问题。

一个《剑指offer》中的经典问题如下：

请设计一个函数，用来判断在一个矩阵中是否存在一条包含某字符串所有字符的路径。

路径可以从矩阵中的任意一个格子开始，每一步可以在矩阵中向左，向右，向上，向下移动一个格子。如果一条路径经过了矩阵中的某一个格子，则该路径不能再进入该格子。

例如： a b c e s f c s a d e e 矩阵中包含一条字符串”bcced”的路径，但是矩阵中不包含”abcb”路径，因为字符串的第一个字符b占据了矩阵中的第一行第二个格子之后，路径不能再次进入该格子。

函数接口：

其中，matrix就是矩阵的字符串（注意是一维数组），str就是目标路径字符串。

思路

要想解决这个问题，首先要理清思路。先把主要流程想出来，再来思考详细内容。

  1. 预留一个bool类型的矩阵，大小与原来的矩阵相同，用来与它形成一一对应的映射关系，用来动态地保存矩阵中的每个格子是否被踩过；

  2. 预留一个int变量下标，用来表示路径字符串需要考虑的位置起点，即下标往左是已经找到的字符串，下标往右是还暂时未找到归属的；

  3. 遍历矩阵中的每个格子，判断每个格子是否符合这样一个条件：从该格子开始能否找出一条包含str的还未找到归属的那部分的路径，只要找到了，就直接返回true ；

  4. 如果遍历完了，表面每个格子都不满足条件，就返回false。

判断从当前格子开始，是否能找出未完成路径

这个函数就是上面的hasPathCore(matrix,rows,cols,i,j,str,pathLength,visited)

其中，i和j是当前遍历的格子的位置，pathLength是已经找到归属的字符串长度，visited是那个bool型矩阵，因此，接下来要考虑的就是当前格子以及未找的归属的字符串部分了。

  1. 如果未找到归属的那部分字符串为’\0’了，说明全部找到了，就直接返回true;
  2. 判断当前格子与未找到归属的第一个字符是否相同，并且当前格子不越界，并且当前格子没有被踩过；
  3. 如果满足上述条件，就把pathLength加一，表示又有一个字符找到了归属，再更新这个格子是否被踩过的信息（即visited数组对应的元素改为true），再判断当前格子的上、下、左、右的相邻格子是否满足接下来的未找到归属的字符串，这里要递归调用本身，只要其中一个满足条件，就满足条件；
  4. 如果不满足条件，就说明该格子不满足条件，别忘了还原：把pathLength减一，这里就是回溯的运用，并更新这个格子是否被踩过的信息，再返回false。

另一个问题（机器人走格子）

地上有一个m行和n列的方格。一个机器人从坐标0,0的格子开始移动，每一次只能向左，右，上，下四个方向移动一格，但是不能进入行坐标和列坐标的数位之和大于k的格子。

例如，当k为18时，机器人能够进入方格（35,37），因为3+5+3+7 = 18。但是，它不能进入方格（35,38），因为3+5+3+8 = 19。请问该机器人能够达到多少个格子？

思路

与上一个问题类似，但是不同的是：这里不是求最优路线，而是求所有路线中，能踩到的总的格子数。要想解决这个问题，首先要理清思路。先把主要流程想出来，再来思考详细内容。

  1. 预留一个bool类型的矩阵，大小与原来的矩阵相同，用来与它形成一一对应的映射关系，用来动态地保存矩阵中的每个格子是否被踩过；
  2. 用原题目中的参数，再加上另外三个参数（当前格子的坐标和那个bool型矩阵）来设计一个函数，返回最终结果（能走过的总格子数），这个函数的意义是，从某个格子为起点，机器人能走过的格子的总数（可以走回头路，但是重复走过的格子只能算作一次）；
  3. 在新函数中，先判断当前格子是否满足题目要求的条件，如果不满足，就返回0；如果满足，把当前位置的bool值改为true，再定义一个count变量，并加上一，再加上上面的格子、下面的格子、左边的格子、右边的格子为起点的能走到的所有格子数的总和，现在这个count就是以当前格子为起点能走到的所有格子的总和了。
  4. 实现满足格子与阈值的关系条件的函数。

plist文件的使用

Plist文件简介与局限

在百度中的定义如下：

Plist是一种文件形式，通常用于储存用户设置，也可以用于存储捆绑的信息，该功能在旧式的Mac OS中是由资源分支提供的。

在游戏开发过程中常常会用到这种文件，因为游戏中有大量的数据信息，比如人物的一些属性：名字、血量、攻击范围等等。这些属性一般会放在一个.plist格式的文件中。

由于该文件一般是在旧式的Mac OS中使用的，那么如果自己用的是Windows系统，怎样在Windows中编辑plist文件呢？
